# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A static website (no framework, no build step to serve) documenting the Catalan physiologist **August Pi-Sunyer** (1879–1965). Served via GitHub Pages on the custom domain in `CNAME` (`www.augustpisunyer.com`). Content is in Catalan (`lang="ca"`).

## Running / previewing

Serve with a real HTTP server (not `file://`, which breaks the PDF `<embed>`s):

```bash
python3 -m http.server 8000   # then open http://localhost:8000
```

There is no lint/test tooling. `scripts/generate.py` (below) is the only build-time step, and it is only needed when publications change.

## Architecture

The site has a **single shared design system** and a **data-driven publication catalog**. Do not reintroduce per-page inline CSS.

- **`css/style.css`** — the whole site's look. Scholarly/archival theme driven by CSS custom properties in `:root` (colors, fonts, spacing). Retune the entire site from there. Every page links this one file.
- **`js/site.js`** — shared behaviour: footer year, home hero play/pause, biography scrollspy, and the bibliography renderer + live search. Feature-detects per page (looks for `#biblio`, `[data-hero-video]`, `.toc`), so one file serves all pages.
- **`data/papers.js`** — **single source of truth** for every publication. A `window.PAPERS` array of records: `{year, authors, title, venue, citation, slug, pdf, available, lang}`. Nothing about publications is hand-maintained anywhere else.
- **`scripts/generate.py`** — reads `data/papers.js` and regenerates (a) the 54 per-paper reader pages and (b) `sitemap.xml`. Run it after any change to `data/papers.js`. Because pages are generated, bibliography links and PDF embeds cannot drift out of sync (the original site's main class of bug).

Page inventory:

- **`index.html`** — landing page. Hero `<video>` (`images/vid_2.mp4`) with overlaid title, an intro, and two entry cards. Hardcoded chrome.
- **`biography.html`** — narrative biography with a sticky table-of-contents, `.bio-section`s, and a chronology `.timeline`. Hardcoded chrome.
- **`bibliography.html`** — thin shell: a search box and an empty `#biblio` mount. `js/site.js` renders the list from `data/papers.js`, **grouped by decade** with **live, accent-insensitive search**. Loads `data/papers.js` before `js/site.js`.
- **`<slug>.html`** (54 generated files, e.g. `1901-la-vida-anaerobica.html`) — one reader page per publication: citation block, PDF `<embed>` (or an "PDF no disponible" notice), download links, and prev/next navigation. **Generated — never edit by hand.**
- **`pdf/<slug>.pdf`** — the source PDFs, named to match their page slug.
- **`images/`** — hero video, portraits, textures. **`sitemap.xml`** — generated by `scripts/generate.py`.

### Naming convention

All HTML pages and PDFs use clean lowercase ASCII slugs: `YYYY-title-words` (e.g. `1913-toxics-anabolics-i-catabolics-del-cor.html` + `pdf/1913-toxics-anabolics-i-catabolics-del-cor.pdf`). No accents, spaces, or parentheses in filenames.

The **interface/chrome** uses the Catalan spelling **"August Pi-Sunyer"** (matching the domain). The **formal citations** inside `data/papers.js` preserve the spelling as originally catalogued, which is **"Pi-Suñer"** (with ñ). This dual spelling is intentional; keep citations verbatim.

### Adding or changing a publication

1. Drop the PDF at `pdf/<slug>.pdf`.
2. Add (or edit) a record in `data/papers.js`. Set `available: false` and `pdf: null` if no PDF exists yet (see the 1919 *Les Vies dels reflexes glucemiants* entry).
3. Run `python3 scripts/generate.py`.

That regenerates the reader page and the sitemap; the bibliography picks up the new record automatically at runtime.
